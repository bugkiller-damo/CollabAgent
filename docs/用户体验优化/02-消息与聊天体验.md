# 消息与聊天体验优化

## 现状问题

| 问题 | 影响 |
|------|------|
| 发送消息无发送中状态 | 网络慢时用户不知是否发送成功 |
| 无消息时间戳分组 | 同用户连续消息重复显示头像和时间 |
| 无 Markdown 渲染 | `**粗体**` 和代码块不渲染 |
| 无表情回复 UI | 只支持文字，不支持 emoji reaction |
| 长消息无折叠 | 超长消息占满屏幕 |
| 无消息编辑/删除 | 发错无法修改 |

## 优化方案

### 优先 P0

1. **发送状态反馈** — 发送中灰色 + 已发送 ✓ + 失败 ❌ 重新发送
2. **消息气泡合并** — 同用户 5 分钟内连续消息合并在一个头像下
3. **Markdown 基础渲染** — 粗体/斜体/代码块/链接（react-markdown）

### P1

4. **Emoji 反应** — 光标悬停消息显示 emoji 选择器，已有 API `/api/messages/:id/reactions`
5. **消息引用回复** — 回复时显示被引用的消息片段
6. **长消息折叠** — 超过 300 字折叠，点击展开

### P2

7. **消息编辑/删除** — 自己的消息可编辑/删除（5 分钟内）
8. **代码高亮** — 代码块用 highlight.js/prism
9. **图片粘贴上传** — Ctrl+V 粘贴图片自动上传
