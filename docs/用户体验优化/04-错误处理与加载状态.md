# 错误处理与加载状态优化

## 现状问题

| 问题 | 影响 |
|------|------|
| API 失败无用户提示 | 消息发不出去用户不知道原因 |
| 页面加载无 spinner | "暂无消息" 可能只是还没加载完 |
| WebSocket 断连无提示 | 用户以为别人没回复，实际是断网了 |
| 表单提交无防重复 | 双击登录按钮发两次请求 |
| 无全局错误边界 | React 崩溃 → 白屏 |

## 优化方案

### 优先 P0

1. **全局 ErrorBoundary** — 组件崩溃时显示"出错了" + 刷新按钮，而非白屏
2. **消息发送 Toast** — 发送失败时顶部红色 Toast "发送失败，点击重试"
3. **WebSocket 连接状态横幅** — 断连时顶部黄色条 "连接中断，重连中..."

### P1（需补后端）

4. **表单提交 Loading** — 登录/注册/发送按钮显示 spinner + 禁用状态
5. **页面骨架屏** — 频道页首次加载显示灰色占位块而非空白
6. **网络超时重试** — fetch 超时 10s 后自动重试 1 次

### P2

7. **离线消息队列** — 断网时消息暂存本地，恢复后批量发送
8. **错误上报** — 前端错误 + API 错误收集到服务器日志
