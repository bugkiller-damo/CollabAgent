## 二、数据模型 (packages/shared)

### 核心实体

| 实体 | 说明 | 关键字段 |
|------|------|---------|
| Message | 消息（频道/DM/线程） | id, channelId, senderId, content, seq, threadId, taskNumber |
| Channel | 频道 | id, serverId, name, visibility(public/private), archived |
| User | 用户 | id, handle, displayName, email, description, tokenVersion |
| Agent | AI代理 | id, name, displayName, runtime, model, status |
| Task | 任务（消息的扩展） | taskNumber, taskStatus, taskAssignee |
| Reminder | 提醒 | fireAt, repeatRule, status, reminderEvents[] |
| Attachment | 附件 | filename, mimeType, sizeBytes, storageKey |

### 消息 Target 格式

```
#channel              → 频道消息
#channel:shortId      → 频道内线程
dm:@peer              → 直接消息
dm:@peer:shortId      → DM内线程
```

### 消息头格式 (RFC 5424-style)

```
[target=#general msg=a1b2c3d4 time=2026-03-15T01:00:00 type=human] @sender: content
```

### WebSocket 消息类型 (14种服务端→客户端)

| 类型 | 说明 |
|------|------|
| agent:deliver | 消息投递 |
| agent:start | 启动agent |
| agent:stop | 停止agent |
| agent:status | agent状态变更 |
| agent:activity_probe | 健康检查 |
| reminder.upsert | 提醒创建/更新 |
| reminder.cancel | 提醒取消 |
| reminder.snapshot | 提醒全量同步 |
| ping/pong | 心跳 |

## 三、后端 (packages/server)

### 3.1 技术栈

- **运行时**: Node.js + TypeScript + Fastify 5
- **数据库**: PostgreSQL 18 + Postgres.js
- **认证**: JWT + bcryptjs + Machine Token
- **实时通信**: WebSocket (ws)

### 3.2 API 路由

| 前缀 | 文件 | 端点 |
|------|------|------|
| `/api/auth` | auth.ts | register, login, refresh, logout-all, profile, change-password, machine-token |
| `/api/channels` | channels.ts | GET /, POST /, members, join, leave, resolve, server |
| `/api/messages` | messages.ts | GET /, POST /send, GET /history, GET /thread/:id, GET /search, reactions |
| `/api/tasks` | tasks.ts | GET /, POST /, claim, unclaim, update-status |
| `/api/reminders` | reminders.ts | schedule, list, cancel, snooze, update, log |
| `/api/profile` | profile.ts | show, update |
| `/api/attachments` | attachments.ts | upload, download |
| `/api/integrations` | integrations.ts | list, login |
| `/api/actions` | actions.ts | prepare |
| `/internal/agent` | agents.ts | GET /, POST /, PATCH /:id, POST /:agentId/send |
| `/api/server/info` | index.ts | 公开端点：频道列表+serverId |
| `/api/health` | index.ts | 健康检查 |
| `/api/agents` | index.ts | Agent管理：创建+列表+自动通知daemon |

### 3.3 数据库 Schema (14张表，自动迁移)

| 表 | 行数 | 说明 |
|-----|------|------|
| users | ~10 | 用户（handle/email/password_hash/token_version） |
| agents | ~3 | AI代理 |
| servers | 1 | 服务器 |
| channels | ~3 | 频道 |
| channel_members | ~5 | 频道成员 |
| messages | ~20 | 消息（支持seq/thread_id/task_*） |
| message_reactions | ~0 | 表情反应 |
| message_attachments | ~0 | 消息附件关联 |
| attachments | ~0 | 附件 |
| reminders | ~0 | 提醒 |
| reminder_events | ~0 | 提醒事件日志 |
| machine_tokens | ~1 | 机器令牌 |
| agent_credentials | ~0 | Agent凭证 |
| action_cards | ~0 | 操作卡片 |

### 3.4 认证体系

| Token 类型 | 前缀 | 用途 |
|-----------|------|------|
| JWT | eyJ... | 用户登录 |
| Machine Token | sk_machine_* | daemon连接 |
| Dev Token | dev-token | 开发模式跳过登录 |

认证中间件（authenticate decorator）支持三种验证：
1. `dev-token` → 直接通过（开发模式）
2. `sk_machine_*` → bcrypt比对machine_tokens表
3. JWT → jwtVerify

### 3.5 数据库自动迁移

启动时读取 `schema.sql`，执行所有 `CREATE TABLE IF NOT EXISTS` + `ALTER TABLE ADD COLUMN IF NOT EXISTS`。安全无副作用，增量追加不丢数据。首次运行时自动种子（1个server + 3个channel）。

### 4.5 WebSocket 实时通信

- 连接: `ws://localhost:5173/ws` → Vite代理到 `ws://localhost:3001/ws`
- 消息路由: agent:deliver → 追加到对应频道 → 实时显示
- 线程回复: 存储到 `thread_{id}` key → 不与主频道消息混淆
- 跨标签页/跨用户同步

## 五、Daemon (packages/daemon)

### 5.1 核心架构

```
DaemonCore
├── WebSocket连接 (ws://server/ws)
├── 消息路由 (@mention → agent)
├── Agent管理（懒加载+按需spawn）
├── ClaudeDriver（持久进程或API fallback）
├── Slock CLI Wrapper（注入认证的批处理脚本）
└── ApiClient（REST API通信）
```

### 5.2 CLI命令 (29个)

| 分组 | 命令数 | 覆盖 |
|------|--------|------|
| auth | 1 | whoami |
| channel | 3 | members, join, leave |
| thread | 1 | unfollow |
| server | 1 | info |
| message | 5 | send, check, read, search, react |
| attachment | 2 | upload, view |
| task | 5 | list, create, claim, unclaim, update |
| profile | 2 | show, update |
| integration | 2 | list, login |
| reminder | 6 | schedule, list, cancel, snooze, update, log |
| action | 1 | prepare |
| **总计** | **29** | |

### 5.3 认证模式 (4种)

| 模式 | Token来源 | 使用场景 |
|------|----------|---------|
| managed-runner | agent-proxy-token-file | daemon本地代理 |
| self-hosted-runner | agent-credential-file | agent自有凭证 |
| legacy-machine | token-file/token-literal | 直接machine token |
| (dev) | dev-token | 开发模式 |

### 5.4 Agent生命周期

```
启动 → loadExistingAgents → Registered (lazy, 不spawn)
  │
收到 @mention
  │
  ├─ agent未spawn → spawnAgent → ClaudeDriver.start()
  ├─ agent已spawn → driver.sendMessage()
  └─ spawn失败 → API fallback (DeepSeek/Claude API)
```

### 5.5 AI回复模式

| 模式 | 触发 | 每次token | 上下文 |
|------|------|----------|--------|
| Claude --print | claude有效 + 安装 | ~24K | 无（单次） |
| API fallback | 默认 | ~500 | 有（对话记忆） |

## 六、安全与认证

- 密码 bcrypt 12轮哈希
- JWT 7天过期（记住我30天）
- Machine token sk_machine_* 单独验证
- Token Version机制：修改密码后所有设备强制重新登录
- 开发模式 dev-token 仅本地可用
