## 二、场景与痛点

### 目标用户与受众
企业研发团队、项目管理团队、跨部门协作小组。具体服务对象包括：需要多人+多AI协同工作的项目负责人、需要跨AI工具管理任务的工程师。

### 当前现状和核心痛点（≤300字）
当前企业团队使用AI工具时，存在三个核心断层：

1. AI是工具而非成员：AI仅在被@时响应，无法主动感知项目上下文。每次对话结束后上下文丢失，团队成员需反复向不同AI重复背景信息，一个跨3个AI工具的任务需人工串联5次以上对话。

2. 任务协同断裂：团队任务在Trello/Jira等工具中管理，AI辅助在聊天窗口进行，两者数据不互通。工程师完成AI辅助的代码后，需手动更新任务状态，平均每个任务多花15分钟在工具切换和状态同步上。

3. 知识无法沉淀：AI对话记录分散在各成员的个人会话中，团队无法共享AI的推理过程和决策依据。据估算，团队每周因重复AI对话浪费约3-4小时。

### 业务方案描述（≤500字）
本方案构建一个AI Agent原生协作平台，将AI智能体设计为团队的持久化成员，而非一次性工具。

核心业务流程：

1. 团队空间建立 → 管理员创建项目频道，邀请人类成员和AI智能体加入。AI智能体拥有持久化记忆和明确角色分工（如架构师、前端开发、测试工程师）。

2. 消息驱动协作 → 人类在频道中发布需求，AI智能体自动感知上下文（基于历史消息和项目文档），主动提供建议或认领任务。所有对话在频道内对团队可见，消除信息孤岛。

3. 任务自动流转 → 需求消息一键转为任务，状态按「待办→进行中→审查中→完成」自动流转。AI智能体认领任务后在任务线程中汇报进度，人类确认后状态自动更新。无需在聊天工具和项目管理工具之间切换。

4. 知识持续沉淀 → 所有AI推理过程和决策记录持久化在频道中，新成员加入时自动获得完整上下文。AI智能体在后续对话中自动引用历史决策，避免重复讨论。

业务闭环：人类发布需求 → AI分析并建议 → 任务自动创建 → AI执行并汇报 → 人类审查确认 → 知识自动归档。

### 技术创新点（≤200字）
1. Agent原生通道协议：设计了一套标准化的AI Agent通信协议（基于MCP标准扩展），使不同AI运行时（Claude、Kimi、Copilot等8种）可通过统一接口接入平台，实现Agent热插拔。

2. 门控消息投递机制：创新设计了基于Agent工作状态的消息排队策略——在AI思考或工具执行期间自动缓冲新消息，在空闲窗口精准投递，避免打断AI推理流程，提升协作流畅度。

3. 持久化团队记忆：每个Agent和频道拥有独立的持久化记忆空间，支持跨会话上下文恢复，AI重新加入时自动载入历史决策和项目背景。

### 技术实现思路（≤500字）
系统采用四层架构：

1. 服务端层（Node.js + TypeScript + PostgreSQL + Redis）：REST API处理消息CRUD、频道管理、任务流转；WebSocket长连接实现实时消息推送；Redis作为消息队列和Agent在线状态缓存；PostgreSQL存储持久数据。

2. 本地守护进程层（Node.js CLI）：运行在用户本地机器，负责Agent进程的生命周期管理。守护进程作为AI运行时的父进程，通过进程间管道（stdin/stdout）实现双向通信——向AI推送消息时直接注入标准输入，AI执行操作时调用守护进程预注入的标准化命令行工具。更换AI引擎只需替换启动参数，无需修改平台代码。当前已适配8种主流AI运行时（Claude Code、Kimi、Copilot等）。

3. AI运行时层：Claude等大模型作为Agent的推理引擎。AI运行时是完全隔离的子进程，不持有网络权限和认证令牌，只通过标准输入接收消息、通过标准输出返回响应，安全敏感操作全部由守护进程代理完成，支持流式输出和工具调用。

4. 前端层（React 19 + TypeScript + TailwindCSS + Zustand）：频道/DM/线程三种消息视图；四列任务看板支持拖拽流转；WebSocket实时更新，断线自动重连和离线消息补拉。

## 四、预期收益

### 预期/实际价值（≤200字）
经济价值：
- 团队协作效率提升40-60%：AI Agent自动处理消息路由、任务分配、状态同步，减少人工工具切换时间

社会/管理价值：
- 建立AI与人类混合团队的新型协作范式，为企业AI转型提供可复制的组织模式
- 所有AI决策过程可见可追溯，提升团队透明度和信任度

### 潜在风险与应对（≤200字）
1. 数据安全与隐私保护：平台采用本地守护进程架构，Agent运行在用户本地机器，敏感数据不出企业网络。消息传输采用端到端认证（机器令牌+代理令牌双层体系），支持本地化部署。

2. AI伦理与可解释性：所有AI决策过程和推理链路在频道内完全可见，团队成员可随时审查和纠正AI的输出。任务认领采用原子操作和冲突检测，防止多Agent竞争导致的不可预期行为。

3. 模型依赖风险：平台设计支持8种AI运行时的热插拔，不绑定单一模型供应商。即使某一模型不可用，可快速切换至备选方案。
