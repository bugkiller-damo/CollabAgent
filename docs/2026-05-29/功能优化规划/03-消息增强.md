# 消息增强

## Markdown 渲染

### 现状
消息以纯文本显示，不支持富文本格式。

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| 基础 Markdown | 粗体/斜体/链接/列表/引用 | 高 |
| 代码语法高亮 | 行内代码 + 代码块（```）语言标注 | 高 |
| 表格渲染 | GFM 表格支持 | 中 |
| 任务列表 | `- [ ]` 和 `- [x]` 渲染 | 低 |

### 实现要点
- 已引入 `react-markdown` + `remark-gfm` + `rehype-highlight`
- 替换消息的 `<p>{msg.content}</p>` 为 `<ReactMarkdown>{msg.content}</ReactMarkdown>`
- 代码高亮需引入 highlight.js CSS 主题

---

## 文件/图片上传

### 现状
无文件上传功能。

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| 拖拽上传 | 拖文件到频道区域自动上传 | 高 |
| 粘贴上传 | Ctrl+V 粘贴剪贴板图片 | 中 |
| 点击上传 | 输入框旁附件按钮 | 中 |
| 图片预览 | 上传后缩略图 + 点击放大（Lightbox） | 高 |
| 文件卡片 | 非图片文件显示：文件名 + 大小 + 下载按钮 | 中 |

### 实现要点
- 监听 `drop` 和 `paste` 事件
- 后端已支持 `/api/attachments/upload`（multipart）
- 上传后获取 attachment ID，随消息发送
- 前端：`AttachmentPreview` 组件
- Lightbox：点击图片 → 全屏查看

## 消息编辑

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| 编辑已发消息 | 仅自己的消息，显示"(已编辑)"标记 | 中 |
| 删除消息 | 软删除（设置 deleted 标志） | 低 |
| 编辑历史 | 保留修改记录 | 低 |

### 实现要点
- 后端：PUT /api/messages/:id → 更新 content + 设置 edited_at
- 前端：编辑模式（消息文本变为 textarea）→ Enter 保存
- 已编辑消息旁显示小字"(已编辑)"

---

## 链接预览

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| URL 自动展开 | GitHub/图片/Twitter 链接自动预览 | 中 |
| OpenGraph 解析 | 后端抓取 og:title/og:image/og:description | 中 |

### 实现要点
- 消息中正则提取 URL
- 后端：GET /api/preview?url=... → 返回 OG 数据
- 前端：`<LinkPreview>` 组件（卡片式）
