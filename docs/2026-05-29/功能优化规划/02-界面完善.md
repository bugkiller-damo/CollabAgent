# 界面完善

## Loading 骨架屏

### 现状
首次加载时页面一片空白或只显示"暂无消息"。

### 优化项
| 场景 | 骨架内容 | 优先级 |
|---|---|---|
| 频道列表加载 | 3 行灰色条模拟频道 | 高 |
| 消息列表加载 | 5 行灰色条模拟消息 | 高 |
| Agent 列表加载 | 3 行卡片骨架 | 中 |
| 设置页加载 | 表单骨架 | 低 |

### 实现要点
- 创建 `<Skeleton>` 组件（宽高 + 动画）
- 各页面在 `loading` 状态时渲染骨架
- 实际数据到达后切换

## 空状态引导

### 现状
空频道显示"暂无消息"一行字，无操作指引。

### 优化项
| 场景 | 引导内容 | 优先级 |
|---|---|---|
| 无频道 | 插图 + "还没有频道，联系管理员创建" | 中 |
| 频道无消息 | 插图 + "发送第一条消息开始对话" | 高 |
| 无 Agent | "创建第一个 AI Agent 来协助工作" + 按钮 | 中 |
| 任务为空 | "将消息转为任务来跟踪进度" | 低 |

### 实现要点
- 创建 `<EmptyState>` 组件（插图 + 文案 + 操作按钮）
- 各类空数据场景统一使用

## 错误边界

### 现状
组件抛出异常 → 整个页面白屏或深蓝色空白。

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| 全局 ErrorBoundary | 捕获未处理异常，显示"出错了" + 重试按钮 | 高 |
| 频道级 ErrorBoundary | 单个频道崩溃不影响侧栏和其他频道 | 中 |
| 错误上报 | 将错误信息记录到 console + 可选后端上报 | 低 |

### 实现要点
- `componentDidCatch` 类组件包裹 `App`
- 显示友好的错误页面："页面遇到问题，[点击重试]"
- 重试按钮调用 `window.location.reload()`

## 主题系统

### 现状
仅暗色主题（bg-gray-900），硬编码。

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| 浅色主题 | 白色背景 + 浅灰边框 | 中 |
| 主题切换按钮 | 侧栏/设置页切换 | 中 |
| 跟随系统 | 自动匹配 OS 的 dark/light 模式 | 低 |

### 实现要点
- Tailwind `dark:` 前缀已支持双主题
- 在 `<html>` 上切换 `class="dark"`
- Zustand `uiStore` 存储主题偏好
- `window.matchMedia('(prefers-color-scheme: dark)')` 检测系统主题

## 响应式适配

### 现状
固定宽度布局，移动端不可用。

### 优化项
| 断点 | 布局 | 优先级 |
|---|---|---|
| < 768px | 侧栏折叠为汉堡菜单 | 中 |
| < 640px | 单栏布局，频道列表占满 | 低 |

### 实现要点
- Tailwind `md:` `sm:` 响应式前缀
- 侧栏用 `transform -translate-x-full` 隐藏
- 汉堡按钮切换侧栏显隐
