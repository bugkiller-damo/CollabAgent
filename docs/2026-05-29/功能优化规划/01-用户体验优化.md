# 用户体验优化

## 消息输入增强

### 现状
输入框仅支持单行文本，Enter 直接发送，无格式辅助。

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| Shift+Enter 换行 | 多行消息支持 | 高 |
| 输入中状态提示 | WebSocket 广播 "正在输入..." | 低 |
| 字符计数 | 超过一定长度显示剩余字数 | 低 |
| 草稿保存 | 未发送的消息自动保存到 localStorage | 中 |

### 实现要点
- ChannelView 的 `<input>` 改为 `<textarea>`，监听 Shift+Enter
- 通过 WebSocket 发送 `user:typing` 事件
- 草稿保存：`localStorage.setItem('draft_#general', content)`

## @提及自动补全

### 现状
输入 `@agent-name` 无提示，需手动记住完整名称。

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| 键入 `@` 弹出列表 | 显示频道内用户和 Agent | 高 |
| 键盘导航 | ↑↓ 选择、Enter 确认、Esc 关闭 | 高 |
| 模糊搜索 | 输入部分名称实时过滤 | 中 |

### 实现要点
- 监听输入框 `@` 字符位置
- 从 `channelStore.members` 和 `/api/agents` 获取候选列表
- 选中后自动补全为 `@handle ` 格式

## 消息操作菜单

### 现状
"回复"按钮始终可见，无其他操作。

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| 悬停显示操作栏 | 鼠标移到消息上显示：回复/反应/更多 | 高 |
| Emoji 反应 | 快速添加 👍👀✅❌ | 中 |
| 复制消息文本 | 一键复制 | 中 |
| 编辑消息 | 仅自己的消息可编辑 | 中 |
| 删除消息 | 仅自己的消息可删除（二次确认） | 低 |

### 实现要点
- 消息行添加 `group-hover` 样式
- 反应面板：已有 reaction 数据表，API 已就绪
- 编辑：PUT /api/messages/:id → 更新 content

## 快捷操作

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| Ctrl+K 频道搜索 | 弹出搜索框，输入即过滤，Enter 跳转 | 中 |
| Ctrl+Enter 发送 | 与换行并存 | 高 |
| Esc 关闭面板 | 关闭线程/设置等侧面板 | 低 |

### 实现要点
- 全局键盘事件监听
- 搜索弹窗组件（Command Palette 模式）
- 冲突处理：输入框聚焦时 Ctrl+K 不触发

## 通知系统

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| 浏览器通知 | 新消息时弹出系统通知 | 中 |
| 未读徽章 | 侧栏频道名后显示未读数 | 高（已有） |
| 音效 | 可选的消息提示音 | 低 |

### 实现要点
- `Notification.requestPermission()` API
- WebSocket 收到消息时判断是否当前频道
- 非当前频道 → 更新未读计数 + 发通知
