# 稳定与性能

## WebSocket 连接管理

### 现状
连接断开显示黄色横幅"连接中断，重连中..."，但无具体诊断。

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| 连接状态指示器 | 侧栏底部绿/黄/红圆点 + 文字 | 高 |
| 重连策略优化 | 指数退避 1s→2s→4s→...→30s | 高（已有） |
| 详细诊断信息 | "无法连接到服务器 (3001 端口未响应)" | 中 |
| Watchdog | 70s 无入站消息 → 主动重连 | 中 |

### 实现要点
- `useWebSocket` hook 已有基本重连逻辑
- 增强状态枚举：connected / reconnecting / disconnected
- 心跳检测：每 30s 发 ping，60s 无 pong → 重连

## 消息可靠性

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| 发送失败重试 | 消息旁红色感叹号，点击重发 | 高 |
| 离线队列 | 断网时消息存入队列，恢复后批量发送 | 中 |
| 去重检查 | 客户端根据消息 ID 去重（已实现） | ✅ |
| 发送确认 | 显示"已发送"/"发送中"/"发送失败"状态 | 中 |

### 实现要点
- `messageStore.sendMessage` 加 try/catch → 失败时设置 `msg.error = true`
- 消息旁显示状态图标
- 点击重试 → 重新发送

## 离线支持

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| 离线消息缓存 | 已加载消息存 IndexedDB/localStorage | 中 |
| 离线提示 | 顶部横幅："你当前处于离线状态" | 中 |
| Service Worker | PWA 离线可用 | 低 |

### 实现要点
- 消息到达时写入 `localStorage.setItem('msgs_#general', JSON.stringify(msgs))`
- 页面加载时先读缓存，再请求新数据
- `navigator.onLine` 检测网络状态
