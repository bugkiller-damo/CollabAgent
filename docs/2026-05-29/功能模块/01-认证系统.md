# 认证系统

## 后端 (`packages/server/src/routes/auth.ts`)

### 端点

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/api/auth/register` | 无 | 注册（handle/email/password） |
| POST | `/api/auth/login` | 无 | 登录（支持 handle 或 email） |
| POST | `/api/auth/refresh` | 无 | Refresh Token 换取新 Access Token |
| POST | `/api/auth/logout-all` | JWT | 所有设备登出（token_version+1） |
| POST | `/api/auth/machine-token` | JWT | 生成 sk_machine_* 机器令牌 |
| PATCH | `/api/auth/profile` | JWT | 修改个人资料 |
| POST | `/api/auth/change-password` | JWT | 修改密码 |
| POST | `/api/auth/profile/password` | JWT | 修改密码（旧路由兼容） |

### 密码规则

- 最少 8 位
- 必须包含字母 + 数字
- bcrypt 12 轮哈希

### Token 机制

| Token | 有效期 | 用途 |
|-------|--------|------|
| Access Token (JWT) | 7天（rememberMe: 30天） | API 认证 |
| Refresh Token | 30天 | 无感刷新 |
| Machine Token | 永久（可 revoke） | Daemon 连接 |
| Dev Token | 开发期 | 跳过登录 |

### 登录安全

- token_version 机制：修改密码 → token_version+1 → 旧 JWT 全部失效
- 错误提示区分：用户不存在 / 密码错误（不泄露用户存在与否）
