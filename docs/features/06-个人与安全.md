# 个人与安全

## 密码安全

### 现状
注册时密码校验"至少 6 位，含字母和数字"，无实时反馈。

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| 密码强度条 | 实时显示弱/中/强（长度+字符类型组合） | 高 |
| 强度规则增强 | 弱: <8位 | 中: 8-11位+字母数字 | 强: ≥12位+大小写+数字+符号 | 中 |
| 密码可见切换 | 👁 按钮切换明文/密文 | 中 |

### 实现要点
- `<PasswordStrength>` 组件：根据规则计算分数
- 颜色条：红(弱) → 黄(中) → 绿(强)
- 后端已有 validatePassword 函数，前端同步规则

## 登录安全

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| 登录限流 | 5 次失败 → 锁定 15 分钟 | 高 |
| 登录设备列表 | 设置页显示："Chrome on Windows · 当前设备" | 中 |
| 踢出其他设备 | 修改密码时自动踢出（token_version 机制已实现） | ✅ |
| 异常登录通知 | 新设备/IP 登录时发通知 | 低 |

### 实现要点
- 后端 rate limit：用 Redis 或内存计数器
- 登录设备表记录：user_id + IP + user_agent + last_seen
- 前端展示设备列表

## Token 安全

### 现状
JWT 存储在 localStorage，存在 XSS 攻击窃取风险。

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| httpOnly Cookie | Token 改为 Cookie 存储，JS 不可读 | 中 |
| CSRF 保护 | 加 CSRF token header | 中 |
| Token 自动刷新 | 过期前 5 分钟自动续期（refreshToken 已实现） | ✅ |
| 会话超时 | 30 分钟无操作 → 自动登出 | 低 |

### 实现要点
- 后端设置 `Set-Cookie: token=xxx; HttpOnly; Secure; SameSite=Strict`
- 前端发请求自动带 Cookie，无需手动管理
- 需要 CSRF token（双重提交 Cookie 模式）

## 账户管理

### 优化项
| 功能 | 说明 | 优先级 |
|---|---|---|
| 头像上传 | 设置页上传/裁剪头像 | 中 |
| 邮箱验证 | 注册后发验证邮件（dev 模式跳过） | 低 |
| 账户注销 | 永久删除账户及所有数据 | 低 |
| 数据导出 | 导出个人消息/文件（GDPR 合规） | 低 |

### 实现要点
- 头像上传：已有 attachment API，上传后设 `avatar_url`
- 邮箱验证：生成 token → 发送邮件 → 点击链接验证
