# 稳定性与部署优化

## 1. 全局错误处理 (P1 · 2h)

**现状**: React 崩溃直接白屏。

**实现**:

| 组件 | 说明 |
|---|---|
| `ErrorBoundary` | 包裹整个 App，崩溃时显示友好错误页 + 重试按钮 |
| `APIErrorHandler` | 统一处理 HTTP 错误码（401→跳登录，500→重试提示） |
| `Toast 通知系统` | 成功/失败/警告消息（复用现有 `toast`） |

## 2. 加载状态与骨架屏 (P1 · 2h)

**现状**: 数据加载中无反馈。

| 场景 | 方案 |
|---|---|
| 频道列表加载中 | 3 个灰色矩形闪烁（skeleton） |
| 消息列表加载中 | 5 行消息骨架 |
| 发送消息中 | 消息旁显示转圈 + "发送中..." |
| 登录中 | 按钮转圈 + "登录中..." |

## 3. API 请求重试 (P1 · 1h)

- 网络错误自动重试 3 次（指数退避）
- 500 错误自动重试 1 次
- 401 不重试，直接跳登录

## 4. 404/403 页面 (P2 · 1h)

- `/404` — 页面不存在
- `/403` — 无权限访问
- 频道不存在 → 显示"此频道不存在或已归档"

## 5. 性能优化 (P2 · 3h)

| 项 | 说明 |
|---|---|
| 虚拟滚动 | 消息列表 > 100 条时启用 `react-virtuoso`（已安装） |
| 图片懒加载 | `loading="lazy"` 属性 |
| 代码分割 | React.lazy + Suspense 按路由分割 |
| Bundle 分析 | Vite 的 `rollup-plugin-visualizer` |

## 6. Docker 一键部署 (P2 · 3h)

```
docker compose up
  ├── postgres (16-alpine)
  ├── server (Node.js + Fastify)
  └── web (nginx + 编译后静态文件)
```

- 编写 `Dockerfile` (server) + `Dockerfile` (web)
- 编写 `docker-compose.yml`
- `.env.example` 环境变量模板

## 7. 健康检查与监控 (P2 · 2h)

- `/api/health` 已有，加 DB 连接检查
- 前端 SystemStatus 组件：显示 server/DB/WS 连接状态
- 错误日志收集（前端 `console.error` 上报到 server）
