# 任务系统优化

> DB Schema 与 API 骨架已就绪，前端 TaskBoard 页面已有。

## 1. 看板完整实现 (P2 · 4h)

**现状**: 四列静态占位，无数据。

**实现**:
- 从 `/api/tasks?channel=xxx` 加载任务
- 四列渲染: `todo | in_progress | in_review | done`
- 任务卡片: 编号、标题、负责人头像、回复数
- 空列提示: "拖拽任务到此处"

## 2. 拖拽交互 (P2 · 3h)

**技术**: `@dnd-kit/core`（已安装）

| 交互 | 说明 |
|---|---|
| 跨列拖拽 | 拖到目标列 → 状态变更 → 乐观更新 |
| 拖拽反馈 | 拖拽中卡片半透明 + 目标列高亮 |
| 冲突回滚 | Claim 冲突时卡片弹回原位 + toast 提示 |
| 批量操作 | 多选卡片 → 批量移动/分配 |

## 3. 任务创建 (P2 · 1h)

| 方式 | 说明 |
|---|---|
| 消息转任务 | 消息旁 "转为任务" 按钮 |
| 快速创建 | 看板顶部输入框回车创建 |
| 批量创建 | 粘贴多行文本，每行一个任务 |

## 4. 任务筛选与排序 (P2 · 2h)

- 按负责人筛选
- 按优先级/标签筛选
- 拖拽排序
- 看板/列表视图切换

## 5. 任务详情面板 (P2 · 2h)

点击任务卡片 → 右侧滑出详情面板:
- 任务描述（Markdown 渲染）
- 负责人 + 修改记录
- 关联的线程讨论
- 附件列表
