# 登录注册体验优化

## 1. 表单交互完善 (P0 · 3h)

**现状**: 登录/注册表单可用，但体验粗糙。

**优化项**:

| 项 | 说明 | 工作量 |
|---|---|---|
| 密码显隐切换 | 密码框加 👁 小眼睛图标 | 30min |
| 实时校验反馈 | 用户名可用性实时检查（失焦触发） | 1h |
| 密码强度条 | 注册时显示密码强度（弱/中/强） | 30min |
| 登录错误细化 | 区分"用户不存在"/"密码错误"/"网络超时" | 30min |
| 邮箱验证提示 | 注册成功后提示"已发送验证邮件" | 30min |

## 2. 第三方登录 (P0 · 4h)

**现状**: 仅支持用户名+密码登录。

**新增**:
- GitHub OAuth 登录
- Google OAuth 登录
- 绑定已有账号（如 handle 冲突）

**技术**: 前端按钮跳转 OAuth → 回调 `/api/auth/oauth/callback` → 服务端签发 JWT

## 3. 密码找回流程 (P1 · 2h)

**现状**: 服务端已有 `/api/auth/forgot-password` + `/api/auth/reset-password`，前端缺页面。

**新增页面**:
- `ForgotPasswordPage` — 输入邮箱 → 发送验证码
- `ResetPasswordPage` — 输入验证码 + 新密码

## 4. 登录态持久化 (P1 · 1h)

**现状**: Token 存 localStorage，刷新不掉。

**优化**:
- 前端启动时调 `/api/auth/me` 恢复用户信息（已有代码，需验证）
- Token 过期前 5 分钟自动刷新
- 多标签页登录态同步（`storage` 事件监听）

## 5. 安全增强 (P2 · 3h)

| 项 | 说明 |
|---|---|
| 登录限流提示 | 连续失败 3 次显示"还剩 X 次尝试" |
| 异地登录提醒 | 新 IP 登录时 WebSocket 推送通知 |
| 二次验证预留 | UI 层面预留 TOTP 输入框接口 |
