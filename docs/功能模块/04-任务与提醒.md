# 任务与提醒

## 任务 (`packages/server/src/routes/tasks.ts`)

### 状态机

```
todo → in_progress → in_review → done
  ↑_________________________________↓ (重新打开)
```

### 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/tasks?channel=...` | 列出频道任务 |
| POST | `/api/tasks` | 批量创建任务 |
| POST | `/api/tasks/claim` | 认领任务（批量+原子检测） |
| POST | `/api/tasks/unclaim` | 放弃任务 |
| POST | `/api/tasks/update-status` | 更新状态 |

### 数据模型

任务复用 messages 表，额外字段：
- `task_number` — 频道内自增编号
- `task_status` — todo/in_progress/in_review/done
- `task_assignee` — 认领者 UUID

### 前端状态

- 路由: `/tasks` + `/tasks/:channelName`
- TaskBoard 页面骨架已存在
- 四列看板（待办/进行中/审查中/已完成）待实现
- @dnd-kit 拖拽待接入

## 提醒 (`packages/server/src/routes/reminders.ts`)

### 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/reminders` | 创建提醒 |
| GET | `/api/reminders` | 列出提醒 |
| DELETE | `/api/reminders/:id` | 取消提醒 |
| POST | `/api/reminders/:id/snooze` | 延迟提醒 |
| PATCH | `/api/reminders/:id` | 更新提醒 |
| GET | `/api/reminders/:id/log` | 事件日志 |

### 重复规则

```
every:15m          — 每 15 分钟
every:2h           — 每 2 小时
daily@09:00        — 每天 09:00
weekly:mon,fri@09:00 — 周一和周五 09:00
```
