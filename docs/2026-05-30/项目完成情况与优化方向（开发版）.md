# CollabAgent 项目完成情况与优化方向（开发版）

> 面向开发者。截至 2026-05-30。CollabAgent = Slock 平台的复刻实现（人机协作团队聊天 + 原生 AI Agent）。
> 配套阅读：`docs/技术架构/Agent-Daemon-完成情况与改造计划.md`（daemon 细节）、`docs/features/`（UX 优化清单）。

---

## 1. 架构总览

pnpm monorepo，4 个 package：

| package | 职责 | 技术栈 |
|---|---|---|
| `shared` | 跨端共享类型 | TypeScript |
| `server` | HTTP + WS 后端 | Fastify 5 · PostgreSQL 18（`postgres` 驱动）· @fastify/jwt · ws · @fastify/multipart · @fastify/static |
| `web` | 前端 | React 19 · Vite 6 · TailwindCSS 3 · Zustand 5 · React Router 7 |
| `daemon` | 本地守护进程 / Agent 运行时 / `slock` CLI | Commander · 本机 Claude（`--print`）· esbuild 打包 CLI |

运行时：server `tsx watch` :3001；web Vite :5173（代理 `/api`、`/files`、`/ws/chat` → 3001）；daemon `pnpm --filter daemon dev`，手动启动，连 WS。
DB：`postgresql://postgres:***@localhost:5432/collabagent`。

数据流：浏览器/agent ↔ server（HTTP + WS 广播）；server ↔ daemon（WS）；daemon 拉起本机 Claude，Claude 通过 `slock` CLI 回调 server 的 `/internal/agent/*` 接口。

## 2. 模块完成情况

### 2.1 前端（web）—— 基本完成
按 `docs/features/01–06` 推进，均已落地：
- **消息体验**：@提及自动补全（含中文名、人/agent）、草稿持久化、输入框自适应、字数；消息操作（复制/回复/👍）；线程回复 + 回复数显示。
- **消息增强**：Markdown 渲染 + 代码高亮（highlight.js）；文件/图片上传（点击/拖拽/粘贴，图片缩略图 + Lightbox，非图片文件卡片）；消息编辑（“(已编辑)”标记，WS 实时同步）；链接预览（OG 卡片，后端带 SSRF 防护）。
- **频道管理**：创建频道弹窗（名称 slugify + 公开/私有）；频道设置（描述/可见性/归档/删除，通用 ConfirmDialog）；成员面板（邀请用户或 agent / 移除 / 角色）；**私有频道访问控制**（可见性过滤 + 读写鉴权）。
- **稳定与性能**：WS 连接状态指示（绿/黄/红）+ 指数退避重连 + 70s watchdog；发送失败重试；离线提示 / 本地缓存 / 离线发送队列；虚拟滚动（`@tanstack/react-virtual`，>100 条启用）。
- **个人与安全**：密码强度条 + 可见切换；登录限流（5 次 → 锁 15 分钟）；头像上传；危险操作二次确认。
- **性能修复**：切频道卡顿（移除逐条 thread 请求、`MessageRow` memo 化）。

### 2.2 后端（server）—— 基本完成
- **认证**：JWT + refreshToken + 机器令牌（`sk_machine_*`）+ token_version 批量失效；登录限流。
- **频道**：CRUD、成员（join/leave/invite/移除/角色）、私有访问控制（`lib/access.ts`）。
- **消息**：send（含附件、线程、@提及自动邀请 agent）、history、search、reactions、编辑、附件聚合返回。
- **任务**：list/create/claim/unclaim/update-status（任务即带 task_number 的消息）。
- **附件**：multipart 上传 + 本地磁盘存储（`lib/storage.ts`，预留 MinIO）+ `/files` 静态服务 + 下载。
- **Agent 接口** `/internal/agent/:id/*`：receive（游标收信）、history、server、channel-members、send、tasks、reactions、search、profile、reminders、upload。
- **提醒**：`reminders` 表 + 调度器（`lib/reminder-scheduler.ts`，20s 轮询，WS 唤醒 daemon）。
- **链接预览**：`/api/preview`（OG 抓取 + SSRF 防护）。
- **WS**：`broadcast`（广播给所有浏览器 + daemon）、`broadcastToDaemons`。

### 2.3 Daemon / Agent 运行时 —— 核心闭环已通
经历从「能说不能听」→ Phase 1（听+说）→ Phase 2（自主运行时）→ Phase 3（能力面）→ P1/P2（记忆/线程/提醒/附件）：
- **身份**：每个 agent 用自己的 `SLOCK_AGENT_ID/TOKEN/SERVER_URL` 启动 Claude，slock 调用以本人身份执行；@提及按已知名匹配（支持中文）。
- **自主模式**：Claude `--print` + 自主系统提示，agent 自己用 `slock` CLI 收发；CLI 用 esbuild 预打包为 `.slock/slock-cli.cjs`（`node` 直跑，回合 ~30s）。
- **持久记忆**：每 agent 一个工作区 `.slock/workspaces/<name>/` + `MEMORY.md`，cwd 指向它，跨会话累积。
- **线程**：被 @ 在线程里 → 在线程内回复（target `#频道:shortid` → 解析 thread_id）。
- **提醒**：到点 server 唤醒 daemon → 起一次 agent 处理。
- **附件**：agent 可 upload 文件并随消息发出。

完成度对照 Slock 见 `Agent-Daemon-完成情况与改造计划.md §9`。

## 3. 已知技术债 / 风险（按影响）

**架构/运行**
1. **延迟 ~30s/回合**：每条消息一次性拉起 Claude（无常驻进程）。优化 = 持久 Claude 进程。
2. **daemon 无生命周期管理**：手动 `pnpm --filter daemon dev` 启动、改代码需手动重启、崩溃不自愈。需服务化（PM2/systemd/Windows 服务）+ watch。
3. **调度器是单实例内存轮询**：多实例会重复触发；daemon 离线时提醒积压。需分布式锁/持久队列。

**安全**
4. **WS 广播未按频道隔离**：`broadcast` 发给所有连接，私有频道内容理论上可被恶意 WS 客户端收到（前端不暴露，但属隔离缺口）。需按频道/成员定向投递。
5. **Agent Claude 用 `--dangerously-skip-permissions`**：有完整 Bash/文件权限。dev 可接受，对外需沙箱/白名单。
6. **Token 存 localStorage**（XSS 风险）；httpOnly Cookie + CSRF 早前评估为高风险改动，未做。

**数据/一致性**
7. **schema 漂移**：`agents`/`reminders` 等表由非 migration 路径建过，迁移文件与实测库有出入；曾出现 `runtime_profile` 双重编码、多处 `*_id → users` 外键对 agent id 不适用（已逐个修，但模式需统一）。建议梳理单一 schema 源 + 重建迁移。
8. **`messages.ts /history` 有历史遗留 bug**：`threadId` 分支里 `p`/`params` 在声明前使用（仅当传 `threadId` 查询参数时触发，前端未用到）。应修。
9. 前端 `AgentManagement` 里 `serverId` 硬编码。

**其它**
10. 无自动化测试 / CI；目前靠 `scripts/smoke-agent.mjs` 手动冒烟 + curl。

## 4. 后续优化方向（建议优先级）

**P0（体验/稳定性硬伤）**
- 持久 Claude 进程，降低回合延迟（30s → 数秒级）。
- daemon 服务化 + 自动重启 + 文件 watch（开发态改完即生效）。
- WS 按频道成员定向投递，关闭私有频道泄露面。

**P1（能力补全，对标 Slock §9 剩余）**
- Action Card（agent 起草、人确认提交）。
- 第三方集成（integration login，给 agent 接外部数据源/工具）。
- DM 私聊（agent 维度）。
- 自主生命周期（常驻、主动巡检，超越「被 @ / 提醒」）。

**P2（工程化/安全收尾）**
- 统一 DB schema 源 + 重建迁移；修 `/history` 的 p/params bug；去掉 serverId 硬编码。
- httpOnly Cookie + CSRF；登录设备列表；账户管理（注销/导出）。
- 调度器持久化/多实例安全。
- 自动化测试 + CI（接口冒烟、关键前端路径）。
- Agent Claude 工具沙箱/权限白名单。

## 5. 验证方式
- 接口冒烟：`node scripts/smoke-agent.mjs [agentId]`（Phase 1+3 全套，14/14）。
- 类型检查：各 package `tsc --noEmit`（server 有少量历史遗留报错，见 §3.8；web/daemon 干净）。
- 前端：`vite build` 通过；浏览器手测（无自动化）。
- daemon：本机有 `claude.cmd` 时端到端可跑；沙箱内只能验证服务端 + 类型/构建。
