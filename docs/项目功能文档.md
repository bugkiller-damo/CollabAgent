> 基于 Slock v0.54.2 逆向分析复现
> 更新时间：2026-05-29

## 一、项目概述

CollabAgent 是一个 AI Agent 原生协作平台，将 AI 智能体设计为团队持久化成员。采用四层架构：服务端 → 本地守护进程 → AI 运行时 → Web 前端。

**技术栈**：Node.js + TypeScript + PostgreSQL + React 19 + Vite + TailwindCSS + Zustand

## 三、模块详情

### 3.1 服务端 (packages/server)

#### 认证系统 (routes/auth.ts)
| 端点 | 方法 | 说明 |
|---|---|---|
| `/api/auth/register` | POST | 邮箱 + 用户名 + 密码注册，自动登录 |
| `/api/auth/login` | POST | 用户名或邮箱登录，返回 JWT + refreshToken |
| `/api/auth/refresh` | POST | refreshToken 换新 JWT |
| `/api/auth/me` | GET | 获取当前用户资料 |
| `/api/auth/profile` | PATCH | 修改昵称/简介/头像 |
| `/api/auth/change-password` | POST | 旧密码验证 → 新密码 |
| `/api/auth/logout-all` | POST | 增加 token_version，所有设备 JWT 失效 |
| `/api/auth/machine-token` | POST | 生成 sk_machine_* 令牌（daemon 认证用） |

**特性**：
- bcrypt 密码哈希（12 轮）
- JWT 签名（支持 token_version 批量失效）
- refreshToken 30 天有效期
- "记住我" 选项控制 accessToken 过期时间
- 机器令牌支持（sk_machine_*）用于 daemon 认证
- 开发模式：`Bearer dev-token` 绕过认证

#### 频道系统 (routes/channels.ts)
| 端点 | 方法 | 说明 |
|---|---|---|
| `/api/channels` | GET | 列出用户加入的频道 |
| `/api/channels` | POST | 创建频道 |
| `/api/channels/:id/members` | GET | 频道成员列表 |
| `/api/channels/:id/join` | POST | 加入频道 |
| `/api/channels/:id/leave` | POST | 离开频道 |
| `/api/channels/resolve` | GET | 解析频道引用（#general / dm:@user） |
| `/api/channels/server` | GET | 服务器信息（频道+agent+成员） |

#### 消息系统 (routes/messages.ts)
| 端点 | 方法 | 说明 |
|---|---|---|
| `/api/messages` | GET | 获取频道消息（公开，支持分页） |
| `/api/messages/send` | POST | 发送消息（支持 threadId） |
| `/api/messages/history` | GET | 消息历史（认证，支持分页） |
| `/api/messages/thread/:id` | GET | 获取线程消息（父消息+回复） |
| `/api/messages/search` | GET | 全文搜索（pg tsvector） |
| `/api/messages/:id/reactions` | POST/DELETE | 添加/删除 emoji 反应 |

**特性**：
- seq 全局递增序列号
- draft/freshness 乐观锁
- 线程支持（thread_id 字段）
- WebSocket 实时广播

#### 任务系统 (routes/tasks.ts)
| 端点 | 方法 | 说明 |
|---|---|---|
| `/api/tasks` | GET | 频道任务列表（支持状态筛选） |
| `/api/tasks` | POST | 批量创建任务 |
| `/api/tasks/claim` | POST | 认领任务（原子操作+冲突检测） |
| `/api/tasks/unclaim` | POST | 释放任务 |
| `/api/tasks/update-status` | POST | 状态流转 |

**状态机**：`todo → in_progress → in_review → done`

#### Agent 管理 (routes/agents.ts)
| 端点 | 方法 | 说明 |
|---|---|---|
| `/api/agents` | GET | 列出所有 Agent（含在线状态） |
| `/api/agents` | POST | 创建 Agent（自动通知 daemon） |
| `/api/agents/:id` | PATCH | 更新 Agent 配置 |
| `/api/agents/channel/:id` | GET | 频道内 Agent 列表 |

#### WebSocket 网关 (ws/handler.ts)
- **双模式**：浏览器客户端 + daemon 客户端
- **认证**：JWT / dev-token / 机器令牌
- **消息类型**：agent:deliver, agent:start, agent:stop, ping/pong
- **广播**：broadcast() → 所有浏览器 + 所有 daemon
- **单播**：sendToDaemon() → 指定 daemon

#### 数据库 (db/schema.sql)
| 表 | 说明 | 关键字段 |
|---|---|---|
| users | 用户 | id, handle, email, password_hash, token_version |
| agents | AI Agent | id, name, runtime_profile, status |
| servers | 服务器 | id, name |
| channels | 频道 | id, name, type(public/private), archived |
| channel_members | 频道成员 | channel_id, member_id, member_type |
| messages | 消息 | id, seq, channel_id, thread_id, task_number |
| message_reactions | 表情反应 | message_id, emoji |
| attachments | 文件附件 | id, filename, storage_key |
| machine_tokens | 机器令牌 | id, token_hash, token_prefix(sk_machine_) |
| reminders | 提醒 | id, fire_at, repeat_rule |

**特性**：启动时自动迁移（schema.sql），种子数据自动插入。

### 3.2 前端 (packages/web)

#### 技术栈
- React 19 + TypeScript + Vite 6
- TailwindCSS 3（暗色主题）
- Zustand 5（状态管理）
- React Router 7
- WebSocket 实时通信

#### 路由表
| 路由 | 组件 | 说明 |
|---|---|---|
| `/login` | LoginPage | 登录页（用户名/邮箱 + 记住我） |
| `/register` | RegisterPage | 注册页（邮箱+用户名+密码确认） |
| `/channels` | → /channels/general | 默认跳转 |
| `/channels/:name` | ChannelView | 频道消息（实时滚动） |
| `/channels/:name/:threadId` | ThreadView | 线程回复 |
| `/tasks` | TaskBoard | 任务看板 |
| `/tasks/:channel` | TaskBoard | 频道任务看板 |
| `/settings/profile` | ProfileSettings | 个人资料 + 改密码 |
| `/admin` | AdminPanel | 管理面板 |
| `/admin/agents` | AgentManagement | Agent 管理（创建/列表/状态） |

#### 状态管理 (9 个 Store)
| Store | 职责 |
|---|---|
| authStore | 用户认证（login/register/logout/refresh/updateProfile） |
| channelStore | 频道列表、未读计数、加入/离开 |
| messageStore | 消息收发、历史拉取、实时追加 |
| taskStore | 任务 CRUD、状态流转 |
| dmStore | DM 会话 |
| profileStore | 用户资料缓存 |
| reminderStore | 提醒管理 |
| integrationStore | 第三方集成 |
| uiStore | 侧栏/主题/布局 |

#### 关键组件
- **AppLayout**：侧栏 + 主内容 + WebSocket 连接
- **Sidebar**：频道列表 + Agent 状态 + 快捷入口（任务/Agent管理）
- **ChannelView**：消息列表 + 输入框 + 自动滚屏 + 线程入口
- **ThreadView**：父消息卡片 + 回复列表 + 回复输入
- **AgentStatusBar**：Agent 在线状态（绿/灰圆点）
- **AuthGuard**：未登录自动跳转 /login

### 3.3 本地守护进程 (packages/daemon)

#### 核心模块
| 模块 | 文件 | 说明 |
|---|---|---|
| DaemonCore | core.ts | WebSocket 连接/重连、消息路由、Agent 生命周期 |
| ApiClient | client.ts | HTTP API 客户端（14 条路径重写） |
| 认证 | auth.ts | 4 种认证模式（managed/self-hosted/legacy ×2） |
| 代理 | proxy.ts | NO_PROXY/WSS_PROXY 支持 |
| ClaudeDriver | drivers/claude.ts | Claude Code 进程管理（spawn/stream-json/工具调用） |
| Probe | drivers/probe.ts | 自动检测 Claude Code 安装位置 |
| System Prompt | system-prompt.md | 28KB 完整 Agent 指令 |
| MCP Bridge | chat-bridge.ts | 15 个 Slock 命令暴露为 MCP tools |
| CLI | cli.ts | 29 个子命令完整实现（Commander.js） |

#### CLI 29 个命令
```
auth whoami
server info
channel members / join / leave
thread unfollow
message send / check / read / search / react
attachment upload / view
task list / create / claim / unclaim / update
profile show / update
integration list / login
reminder schedule / list / cancel / snooze / update / log
action prepare
```

#### Agent 生命周期
```
创建 Agent (UI) 
  → server POST /api/agents 
  → broadcast agent:start 到 daemon
  → daemon 注册 agent（懒加载）

@agent 消息 
  → daemon 收到 agent:deliver
  → findMentionedAgent
  → spawnAgent (首次) 或 sendToAgent (后续)
  → Claude --print → 解析输出 → sendReply
```

## 四、部署与运维

### 环境要求
- Node.js 22+
- PostgreSQL 16+
- pnpm 9+
- Claude Code 2.1+（可选）

### 启动命令
```powershell
# 终端 1：数据库（一次性）
$env:PGPASSWORD = "P@ssw0rd"
psql -U postgres -c "CREATE DATABASE collabagent;"

# 终端 2：服务端
pnpm install
pnpm --filter server dev         # http://localhost:3001

# 终端 3：前端
pnpm --filter web dev            # http://localhost:5173

# 终端 4：daemon（可选）
pnpm --filter daemon dev -- --server-url http://localhost:3001 --api-key sk_machine_xxx
```
